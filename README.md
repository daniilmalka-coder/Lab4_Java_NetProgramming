# Metro Card System (TCP Client-Server)

Ця лабораторна робота демонструє реалізацію багатоклієнтської системи обліку метро-карток за допомогою Java Sockets (TCP).

## Опис проекту
Проект побудований на архітектурі **Клієнт-Сервер**. Сервер здатний обробляти запити від кількох клієнтів одночасно завдяки використанню багатопотоковості (`Threads`). Обмін даними відбувається через об'єктну серіалізацію.

### Основні можливості:
- Реєстрація нових метро-карток.
- Поповнення балансу картки.
- Перевірка поточного балансу.
- Автоматичне логування всіх підключень на стороні сервера.

## Структура пакетів
- `models` — класи даних (`User`, `MetroCard`, `MetroCardBank`).
- `network` — протокол взаємодії (`OpType` enum та `MetroRequest`).
- `server` — серверна логіка та обробники клієнтів.
- `client` — клієнтська частина для надсилання запитів.

## Як запустити

1. **Запуск сервера**:
   - Відкрийте клас `server.MetroServer` і запустіть його.
   - Ви побачите повідомлення: `Metro Server started`.

2. **Запуск клієнта**:
   - Відкрийте клас `client.Client` і запустіть його.
   - Ви можете запустити кілька екземплярів клієнта одночасно для тестування багатопотоковості.

## 🛠 Технології
- **Java SE 8+**
- **TCP Sockets** (java.net)
- **Object Serialization** (java.io.Serializable)
- **Multi-threading**
